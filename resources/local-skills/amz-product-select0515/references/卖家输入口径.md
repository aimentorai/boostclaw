# 卖家输入口径

## 用途

这份参考用于把卖家的自然表达，映射成更稳定的内部判断结构。

重点不是让用户学会字段名，而是让 Skill 能听懂卖家的说法。

## 输入映射

- “你想做哪个站点” -> 市场
- “你现在是想看关键词、拆竞品，还是按条件找产品” -> 分析入口
- “你在看哪个词 / 哪个竞品 / 哪个方向” -> 核心输入
- “你想做多少钱的产品” -> 价格带
- “你希望这个产品卖到什么量级” -> 目标销量预期
- “你能接受多卷的市场” -> 竞争容忍度
- “你首批准备投多少钱” -> 首批预算
- “你想不想优先做轻小件” -> 重量和体积偏好
- “你有没有工厂、材质、工艺或交付优势” -> 供应链优势
- “哪些类目你明确不做” -> 排除类目
- “要不要直接排除认证复杂的产品” -> 认证偏好
- “要不要排除季节性太强的产品” -> 季节性偏好

## 推荐追问方式

### 用户方向还很模糊时

- “你现在是已经有方向了，还是想让我从头帮你找机会？”
- “你更偏向稳一点的，还是愿意冒一点风险换更低竞争？”

### 用户只给了一个词时

- “你是想判断这个词能不能做，还是想顺着这个词继续挖产品机会？”
- “你更关注需求、竞争，还是利润空间？”

### 用户只给了一个竞品时

- “你是想看它为什么卖得好，还是想判断你自己适不适合跟？”
- “你更在意流量词、利润空间，还是供应链可复制性？”

### 用户按条件找产品时

- “预算大概在哪个区间？”
- “你有没有明确不碰的类目？”
- “你是想优先做轻小件、低认证、好补货的品吗？”

## 沟通原则

- 能用口语就不用字段名
- 能问区间就不逼用户给精确值
- 能从上下文推断的，不反复追问
- 如果信息不足，明确写出假设条件
