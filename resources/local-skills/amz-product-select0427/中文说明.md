# 亚马逊选品 Skill 中文说明

## 这份 Skill 是干什么的

这份 Skill 是给做亚马逊的中文卖家准备的。

说白了，它不是拿来讲技术逻辑的，而是帮你更快判断一个方向能不能做。你可以拿它看关键词，拆竞品，也可以按自己的预算、供应链和选品偏好去筛产品。它更像一个懂跨境的人陪你一起过一遍思路，而不是让你填一堆参数。

## 它主要帮你解决什么问题

很多时候不是完全没方向，而是手里有一点线索，但不好判断。

比如你已经看到一个词，不确定是不是太卷；或者刷到一个竞品，想知道它到底是产品本身有机会，还是只是品牌、内容、投放做得强；再或者你预算有限，只想先找轻一点、小一点、别太麻烦的品，这时候就可以用它来先筛一轮。

## 它的说话方式

我写这份 Skill 时，故意没让它走“系统风”。

它不会一上来跟你说什么字段名、模式名、评分卡，也不会甩一堆看起来很完整、其实不太好用的术语。正常情况下，它会直接按卖家的说法来接你的话，然后给你一个比较像实战判断的回复。

比如它更适合这样说：

- 这个词不是不能做，问题是后面容易卷价格
- 这个竞品能跑起来，不一定代表你现在适合跟
- 如果你现在预算不高，先从轻小件切会更稳一点

## 你可以怎么用

你后面直接用自然一点的话去叫它就行，不需要刻意“像在调用系统”。

常见的用法大概就是这几种：

- 帮我看这个关键词能不能做
- 帮我拆一下这个竞品值不值得跟
- 我预算 5 到 10 万，想找轻小件、不要认证太复杂的产品
- 我有华南这边的塑料供应链，帮我看什么方向更适合我
- 帮我先排掉强季节性、太重、要做认证的类目

## 它内部大概怎么判断

虽然表面上是用卖家口吻在聊，但底层判断还是有框架的，只是我没有把它写得太硬。

它一般会看几件事：这个需求到底稳不稳，这个市场是不是已经卷得太厉害，做出来之后有没有利润空间，会不会压货，你现在的资源能不能接得住，还有最大的坑是在认证、侵权、季节性，还是同质化。

也就是说，它不是只看“这个品有没有人卖”，而是会顺手帮你判断“这个品适不适合现在的你去做”。

## 它会用到哪些 MCP 接口

这份 Skill 不是每次都要走 MCP。

如果你只是想聊选品思路、判断逻辑、排雷框架，那它可以直接按规则帮你分析；但只要你问到“现在这个词有没有量”“这个 ASIN 最近卖得怎么样”“这个类目这 30 天是在涨还是在掉”这种实时问题，它就应该走真实数据。

这里我已经改成统一规则：

- 只要是 Amazon 实时数据，一律从 `proboost-Amazon-mcp` 取
- 不再默认用别的 Amazon 数据源来替代
- 如果当前会话里没加载出这个 MCP，就明确说数据源不可用，不编数字

如果你问“这份 Skill 到底用了哪些 MCP”，可以直接理解成两层：

- 第一层是先确认当前会话里有没有加载出 `proboost-Amazon-mcp`
- 第二层是真要查实时 Amazon 数据时，就调用 `proboost-Amazon-mcp`

它会优先去拿这几类数据：

- 关键词需求
- ASIN / Listing 反查
- 类目或子类目趋势
- 价格带、BSR、评论、评分、卖家数量
- 7 天 / 30 天 / 90 天变化

具体工具名我没有写死在正文里，因为 MCP 在不同会话里暴露出来的函数名可能不完全一样。更稳妥的做法是先动态发现，再调用最合适的那个。

## 目录里放了什么

这个文件夹里最重要的是下面几份：

- `SKILL.md`：主文件，决定这份 Skill 什么时候触发、怎么追问、最后怎么输出
- `references/卖家输入口径.md`：把卖家的自然表达和内部判断逻辑对起来
- `references/判断框架.md`：选品时主要看哪些点
- `references/输出模板.md`：最后怎么把结果说得更像一个做业务的人
- `references/MCP接口说明.md`：什么时候该走真实 Amazon 数据，以及优先查哪些接口

## 如果你后面还想继续改

这版已经能用了，但如果你后面想把它做得更贴近自己的业务，最值得补的不是再加更多术语，而是加你自己的经验判断。

比如哪些类目你默认就不碰，哪些站点你更熟，什么样的价格带对你更舒服，什么类型的产品你一看就知道后面售后麻烦。这些东西一旦补进去，这份 Skill 会明显更像“你自己的选品助手”，而不是一个通用模板。
