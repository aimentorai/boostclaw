# 判断框架

## 用途

这份参考用于帮助 Skill 在内部保持判断稳定，但对外表达仍然保持卖家口吻。

## 1. 需求够不够

重点判断：

- 是不是有人持续在搜、在买
- 是长期需求，还是短期热点
- 出单空间是不是太小

推荐表达：

- “这个方向不是没人搜，需求是有的。”
- “这个需求更像短期波动，不太像长期稳定款。”

## 2. 市场卷不卷

重点判断：

- 头部是不是已经被强品牌卡住
- 评论量门槛高不高
- 广告位是不是很挤
- 同质化是不是很严重

推荐表达：

- “这个市场有需求，但已经比较卷了。”
- “头部还没有完全被大品牌锁死，说明还有切进去的空间。”

## 3. 做出来赚不赚

重点判断：

- 价格带是否合理
- 运费、仓储和退货压力大不大
- 会不会很压首批资金
- 后面会不会容易陷入价格战

推荐表达：

- “这个价格带有利润空间，但要看你能不能把采购和物流打下来。”
- “问题不在卖不动，问题在于压货和利润会被吃掉。”

## 4. 你适不适合做

重点判断：

- 用户有没有供应链优势
- 团队能不能承接这个复杂度
- 是否适合现在这个阶段切入

推荐表达：

- “如果你现在预算不高，又想先做轻小件，这类产品更适合你。”
- “这个方向不一定差，但对你当前阶段来说偏重了。”

## 5. 最大的坑在哪

重点判断：

- 认证复杂
- 侵权风险
- 强季节性
- 售后复杂
- 易碎、易坏、易退货

推荐表达：

- “这个方向最大的坑不是需求，而是认证门槛。”
- “它的问题不是卖不动，而是太容易陷入同质化和价格战。”

## 6. 结论等级

默认用下面三档：

- 这个方向可以重点看
- 这个方向可以小测
- 这个方向先别急着做

除非用户明确要求，不要默认输出生硬的分数表。
